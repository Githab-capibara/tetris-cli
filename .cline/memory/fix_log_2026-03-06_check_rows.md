## Проблема
В функции `check_rows()` (строки 668-675) алгоритм удаления заполненных линий работал некорректно при удалении нескольких линий одновременно.

### Исходный код (некорректный):
```rust
for (shift_count, &row) in rows_to_remove.iter().rev().enumerate() {
    for y in (0..row).rev() {
        self.blocks[y + shift_count + 1] = self.blocks[y];
    }
    self.blocks[shift_count] = [-1; GRID_WIDTH];
}
```

### Описание ошибки:
- При удалении нескольких строк одновременно, сдвиг происходил некорректно
- `self.blocks[shift_count]` очищал неправильные строки
- После первого сдвига данные в массиве изменялись, что приводило к ошибкам при последующих итерациях

## Решение
Заменён алгоритм на корректный с использованием временного массива:

```rust
let mut new_blocks = [[-1; GRID_WIDTH]; GRID_HEIGHT];
let mut write_idx = GRID_HEIGHT - 1;

for read_idx in (0..GRID_HEIGHT).rev() {
    if !rows_to_remove.contains(&read_idx) {
        new_blocks[write_idx] = self.blocks[read_idx];
        if write_idx > 0 {
            write_idx -= 1;
        }
    }
}

self.blocks = new_blocks;
```

### Преимущества нового алгоритма:
- Копируются только незаполненные строки
- Сдвиг происходит корректно независимо от количества удалённых линий
- Верхние строки автоматически остаются пустыми (-1)
- Нет конфликтов при чтении и записи в один массив
