## Критические ошибки (исправлено 3/3)

### 1. src/game.rs:922 - SystemTime::now().expect()
**Проблема:** expect() может паниковать если время меньше UNIX_EPOCH
**Решение:** Замена expect() на unwrap_or(Duration::ZERO)
**Строки:** 1020-1024

### 2. src/main.rs:372 - terminal_size().expect()
**Проблема:** expect() может паниковать если не удалось получить размер терминала
**Решение:** Обработка через match с возвратом из функции и сообщением об ошибке
**Строки:** 386-395

### 3. src/io.rs - 9 мест с .expect()
**Проблема:** 9 вызовов expect() могут вызвать панику
**Решение:** Замена всех expect() на if let Err() с eprintln! и возвратом/exit
**Исправленные методы:**
- Canvas::new() - 4 expect()
- Canvas::reset() - 2 expect()
- Canvas::draw_strs() - 1 expect()
- Canvas::draw_string() - 1 expect()
- Canvas::flush() - 1 expect()

## Серьёзные ошибки (исправлено 6/6)

### 1. src/game.rs:680-688 - Soft Drop очки
**Проблема:** Очки могли начисляться без фактического перемещения
**Решение:** Добавлена проверка что перемещение произошло (prev_y comparison)

### 2. src/game.rs:736-740 - Проверка проигрыша
**Проблема:** Проверка только позиции Y фигуры, не блоков

### 3. src/game.rs:1006 - Clone для ghost фигуры

### 4. src/io.rs:193 - get_key() возвращает 0
**Проблема:** Возврат 0 при ошибке без обработки
**Решение:** Уже реализовано через match, добавлены комментарии

### 5. src/highscore.rs:23-35 - DefaultHasher
**Проблема:** DefaultHasher не криптографически стойкий
**Решение:** Добавлен комментарий о безопасности в документацию модуля

### 6. src/tetromino.rs:73-82 - thread_rng()
**Проблема:** thread_rng() создаётся при каждом заполнении мешка
**Решение:** rng кэшируется в структуре BagGenerator, используется один экземпляр
