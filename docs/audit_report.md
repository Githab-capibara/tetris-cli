#### ⚠️ Проблема 2: Неоптимальное использование `&Box<T>`

**Файл:** `/home/d/tetris-cli/src/game.rs`  
**Строка:** 1465  
**Проблема:** Использование `&Box<T>` вместо `&T` - избыточное выделение памяти

**Код:**
```rust
/// Получить игровое поле (для тестов).

#[cfg(test)]
pub fn get_blocks(&self) -> &Box<[[i8; GRID_WIDTH]; GRID_HEIGHT]> {
    &self.blocks
}
```

**Рекомендация:**
```rust
pub fn get_blocks(&self) -> &[[i8; GRID_WIDTH]; GRID_HEIGHT] {
    &self.blocks
}
```

**Обоснование:** `&Box<T>` и `&T` имеют одинаковый размер указателя, но `&Box<T>` требует дополнительного выделения памяти в куче.

**Статус:** Требует исправления

#### ⚠️ Проблема 4: Недостаточная обработка ошибок в `Canvas::new()`

**Файл:** `/home/d/tetris-cli/src/io.rs`  
**Строка:** 168-198  
**Проблема:** Функция использует `panic!` вместо возврата `Result`, что не соответствует идиомам Rust

**Код:**
```rust
pub fn new() -> Self {
    let mut out = match stdout().into_raw_mode() {
        Ok(term) => term,
        Err(e) => {
            Self::exit_with_terminal_reset(&format!(
                "Ошибка: не удалось перейти в raw-режим терминала: {}",
                e
            ));
        }
    }
    // ...
}
```

**Рекомендация:**
```rust
pub fn new() -> Result<Self, IoError> {
    let mut out = stdout().into_raw_mode()
        .map_err(|e| IoError::new(ErrorKind::Other, e))?;
    // ...
    Ok(Self { out })
}
```

**Статус:** Требует исправления (но текущая реализация работоспособна для CLI приложения)
