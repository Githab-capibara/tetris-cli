## 📊 Обзор архитектуры

Tetris CLI использует **модульную архитектуру** с чётким разделением ответственности между компонентами. Проект следует принципам **SOLID** и **DRY**, обеспечивая поддерживаемость и расширяемость кода.

### Ключевые принципы

- **Модульность**: каждый модуль отвечает за одну область функциональности
- **Инкапсуляция**: скрытие внутренней реализации, публичный API через mod.rs
- **Безопасность**: защита данных через хеширование и валидацию
- **Производительность**: 60 FPS стабильно через оптимизированный игровой цикл
- **Тестируемость**: 1500+ модульных и интеграционных тестов
- **Отсутствие циклических зависимостей**: общие типы вынесены в отдельный модуль types.rs

## 📦 Модули проекта

### main.rs

**Назначение**: Точка входа в приложение и управление состоянием меню.

**Основные компоненты**:
- `MENU` — массив строк для отображения главного меню
- `LEADERBOARD_MENU` — меню таблицы лидеров
- `main()` — инициализация терминала, загрузка рекорда, цикл меню
- `get_player_name()` — запрос имени для таблицы лидеров
- `show_leaderboard()` — отображение топ-5 рекордов
- `show_game_stats()` — показ статистики после игры

**Поток выполнения**:
1. Загрузка рекорда из конфигурации
2. Загрузка и валидация таблицы лидеров
3. Проверка размера терминала
4. Инициализация Canvas и KeyReader
5. Цикл отрисовки меню (60 FPS)
6. Обработка нажатий:
   - `Enter` — запуск классической игры
   - `R` — запуск режима спринт
   - `M` — запуск режима марафон
   - `L` — отображение таблицы лидеров
   - `Backspace` — выход
7. После игры: отображение статистики, сохранение рекорда

### game/ (модуль)

**Назначение**: Основная игровая логика Tetris.

**Структура модуля**:

```
src/game/
├── mod.rs       # Публичный API, re-export, impl GameState
├── state.rs     # Структуры данных, константы
├── logic.rs     # Игровая логика, физика, коллизии
├── scoring.rs   # Система очков, уровни, комбо
└── render.rs    # Отрисовка, анимации
```

#### state.rs

**Назначение**: Структуры состояния игры и константы.

**Основные компоненты**:
- `GameState` — основное состояние игры (поля публичные для доступа из подмодулей)
- `GameStats` — статистика игры (время, фигуры, комбо)
- `GameMode` — режим игры (Classic, Sprint, Marathon)
- `Achievement` — система достижений (заготовка)
- Константы: `FPS`, `INITIAL_FALL_SPD`, `LINE_SCORES`, `COMBO_BONUS`, и др.

**Пример использования**:
```rust
use tetris_cli::game::GameState;

let mut game = GameState::new();
game.play(&mut cnv, &mut inp, &high_score_display);
```

#### logic.rs

**Назначение**: Игровая логика, физика, обработка ввода.

**Основные функции**:
- `update()` — обновление состояния игры за кадр
- `handle_input()` — обработка ввода пользователя
- `handle_falling()` — логика падения фигур
- `can_move_curr_shape_direction()` — проверка возможности движения
- `can_rotate_curr_shape()` — проверка возможности вращения
- `rotate_with_wall_kick()` — вращение с проверкой столкновений
- `save_tetromino()` — фиксация фигуры в сетке
- `check_rotation_collision()` — проверка столкновений при вращении

**Wall Kick система**:
- Используется `WALL_KICK_OFFSETS` для проверки смещений
- 5 позиций для проверки возможности вращения
- Super Rotation System (SRS) — упрощённая реализация

#### scoring.rs

**Назначение**: Система очков, уровней и комбо.

**Основные функции**:
- `find_full_rows()` — поиск заполненных линий (битовая маска `u32`)
- `remove_rows()` — удаление заполненных линий
- `update_score_and_level()` — обновление очков и уровня
- `handle_landing()` — обработка приземления фигуры
- `handle_hold()` — удержание фигуры
- `handle_hard_drop()` — жёсткое падение
- `handle_soft_drop()` — мягкое падение

**Система очков**:
```rust
pub const LINE_SCORES: [u128; 4] = [
    100,  // 1 линия: 100 × 2^0 = 100
    200,  // 2 линии: 100 × 2^1 = 200
    400,  // 3 линии: 100 × 2^2 = 400
    1800, // 4 линии (Tetris): 100 × 2^3 + 1000 = 1800
];
pub const COMBO_BONUS: u128 = 50; // Бонус за комбо
pub const LEVEL_BONUS_MULT: u128 = 500; // Множитель уровня
```

**Оптимизация**:
- Битовая маска `u32` для хранения заполненных линий
- Уменьшение памяти с 20 байт до 4 байт
- O(n) сложность где n = `GRID_HEIGHT` (20 итераций)

#### render.rs

**Назначение**: Отрисовка игрового поля и UI.

**Основные функции**:
- `draw()` — отрисовка текущего состояния
- `check_rows()` — проверка и анимация заполненных линий
- `animate_clear()` — анимация очистки линий
- `draw_ghost_shape()` — отрисовка призрачной фигуры
- `draw_next_shape()` — отрисовка следующей фигуры
- `draw_held_shape()` — отрисовка удержанной фигуры
- `draw_ui()` — отрисовка интерфейса (счёт, уровень, линии)

**Константы отрисовки**:
```rust
pub const PREVIEW_X: u16 = 24;  // Позиция предпросмотра по X
pub const PREVIEW_Y: u16 = 8;   // Позиция предпросмотра по Y
pub const HOLD_PREVIEW_X: u16 = 2;  // Позиция удержания по X
pub const HOLD_PREVIEW_Y: u16 = 8;  // Позиция удержания по Y
```

### tetromino.rs

**Назначение**: Определение типов фигур и их поведения.

**Типы фигур (ShapeType)**:

| Тип | Название | Цвет | Координаты |
|-----|----------|------|------------|
| T | T-образная | Пурпурный (Magenta) | (-1,0), (0,0), (1,0), (0,1) |
| L | L-образная | Жёлтый (Yellow) | (-1,-1), (0,-1), (0,0), (0,1) |
| J | J-образная | Синий (Blue) | (1,-1), (0,-1), (0,0), (0,1) |
| S | S-образная | Зелёный (Green) | (0,-1), (0,0), (1,0), (1,1) |
| Z | Z-образная | Светло-красный (LightRed) | (0,-1), (0,0), (-1,0), (-1,1) |
| O | Квадрат | Светло-жёлтый (LightYellow) | (0,0), (1,0), (0,1), (1,1) |
| I | Линия | Голубой (Cyan) | (0,-1), (0,0), (0,1), (0,2) |

**Bag Generator**:
- Система 7-bag гарантирует равномерное распределение фигур
- Каждые 7 фигур содержат все 7 типов
- Использует алгоритм Fisher-Yates для перемешивания
- **Оптимизация**: используется `reserve()` + `extend_from_slice()` для уменьшения аллокаций

### highscore.rs

**Назначение**: Сохранение и загрузка рекордов с защитой от подделки.

**Структура SaveData**:
```rust
pub struct SaveData {
    pub high_score: u64,           // Значение рекорда
    pub high_score_salt: String,   // Случайная соль для хеширования
    pub high_score_hash: String,   // Хеш рекорда с солью
}
```

**Структура LeaderboardEntry**:
```rust
pub struct LeaderboardEntry {
    pub name: String,    // Имя игрока
    pub score: u64,      // Значение рекорда
    salt: String,        // Соль (приватное)
    pub hash: String,    // Хеш записи
}
```

**Защита от подделки**:
- BLAKE3 хеширование
- Соль для хешей (32 байта)
- Валидация имён (whitelist)
- Rate limiting (10 записей в минуту)
- Integrity check при загрузке

## 🎮 Игровой цикл (детали)

### Частота обновления
- **FPS**: 60 кадров в секунду
- **Интервал кадра**: ~16.67 мс (`FRAME_DELAY_MS`)

### Физика падения
- Начальная скорость: 0.9 блоков/секунду
- Прирост скорости: +0.05 за каждую заполненную линию
- Плавное движение с интерполяцией по времени

### Таймер приземления
- Задержка после касания: 0.1 секунды
- Даёт время на перемещение фигуры перед фиксацией

### Wall Kick
- Система проверки столкновений при вращении
- Константа `WALL_KICK_OFFSETS` содержит смещения
- 5 позиций для проверки возможности вращения

## 🔒 Безопасность

### Защита рекордов

1. **BLAKE3 хеширование** — криптографически стойкая функция
2. **Соль для хешей** — 32 байта случайных данных из OS RNG
3. **Валидация имён** — фильтрация опасных символов
4. **Path traversal защита** — в `controls.rs`
5. **Rate limiting** — cooldown 10 секунд между записями
6. **Integrity check** — проверка хеша при загрузке
7. **Массив на стеке** — `blocks` размещён в куче через `Box`

### Инкапсуляция

- Приватные поля в `LeaderboardEntry` (salt)
- Публичные геттеры для контролируемого доступа
- Валидация всех входных данных

### Проверки границ

- `debug_assert!` для безопасности cast в `usize`
- `.max(0.0)` для предотвращения отрицательных значений
- Проверки границ массивов перед доступом
