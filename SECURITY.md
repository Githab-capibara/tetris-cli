## 🚨 Сообщение об уязвимостях

### Критические уязвимости

Для сообщений о критических уязвимостях (безопасность, потеря данных):

**Вариант 1: GitHub Private Vulnerability Reporting**

Используйте [GitHub Security Advisories](https://github.com/Githab-capibara/tetris-cli/security/advisories) для создания приватного отчёта.

**Вариант 2: Email**

Отправьте описание уязвимости на email (если указан в профиле мейнтейнера).

**Вариант 3: GitHub Issues (не рекомендуется)**

Создайте issue с пометкой **[SECURITY]** в заголовке, но **не раскрывайте детали уязвимости** в публичном доступе.

### Что включать в отчёт

**Минимальная информация:**
```markdown
### Тип уязвимости
[Описание типа уязвимости]

### Затронутые версии
[Версии, которые подвержены уязвимости]

### Воспроизведение
[Шаги для воспроизведения (приватно)]

### Возможные последствия
[Что может произойти при эксплуатации]

### Предлагаемое решение (если есть)
[Ваши предложения по исправлению]
```

**Дополнительно:**
- Скриншоты/логи (приватно)
- Доказательство концепции (PoC)
- Ссылки на похожие уязвимости (CVE)

### Время ответа

- **Критические**: 24-48 часов
- **Высокие**: 3-5 дней
- **Средние**: 1-2 недели
- **Низкие**: 2-4 недели

## 🛡️ Безопасность в проекте

### Текущие меры безопасности

#### 1. Защита рекордов

**BLAKE3 хеширование с солью:**
```rust
// Генерация криптографически стойкой соли
let salt = get_random_hash(); // 32 байта случайных данных

// Хеширование с солью
let hash = blake3::hash(&format!("{}{}", salt, score));

// Проверка целостности через verify_and_get_score()
match save_data.verify_and_get_score() {
    Some(score) => score, // Рекорд валиден
    None => {
        // Обнаружена подделка, запись в лог
        eprintln!("Предупреждение: попытка подделки рекорда!");
        0
    }
}
```

**Защита от подделки:**
- Соль генерируется через `rand::thread_rng()` (криптостойкий)
- BLAKE3 — криптографически стойкая хеш-функция
- Проверка через `verify_and_get_score()` при каждой загрузке
- Логирование попыток подделки рекорда

#### 2. Валидация входных данных

**Whitelist валидация имён в таблице лидеров:**
```rust
// Фильтрация символов через whitelist
let validated: String = trimmed
    .chars()
    .filter(|&c| is_valid_name_char(c)) // Только разрешённые символы
    .take(20) // Ограничение длины
    .collect();

// Проверка разрешённых символов
fn is_valid_name_char(c: char) -> bool {
    c.is_alphanumeric() || matches!(c, '_' | '-' | ' ' | '.')
}
```

**Разрешённые символы (whitelist):**
- Буквы (a-z, A-Z, Unicode)
- Цифры (0-9)
- Специальные: `_`, `-`, ` `, `.`

**Запрещены все остальные символы** — подход whitelist обеспечивает максимальную защиту.

#### 3. Path Traversal защита

**Усиленная валидация путей в `controls.rs` с `canonicalize()`:**
```rust
// Усиленная защита с canonicalize()
let canonical_path = path.canonicalize()
    .map_err(|_| "Недействительный путь")?;

// Проверка что путь находится в разрешённой директории
let allowed_dir = std::env::current_dir()?;
if !canonical_path.starts_with(&allowed_dir) {
    return Err("Path traversal запрещён".into());
}

// Дополнительная проверка на абсолютные пути
if path.starts_with('/') {
    return Err("Абсолютные пути запрещены".into());
}

// Проверка на path traversal (..)
if path.components().any(|c| c.as_os_str() == "..") {
    return Err("Path traversal запрещён".into());
}

// Сохранение только относительных путей
File::create(&path)?;
```

**Запрещено:**
- Абсолютные пути (начинаются с `/`)
- Path traversal (`..`)
- Символические ссылки на файлы вне разрешённой директории
- Пути, выходящие за пределы разрешённой директории

**Многоуровневая защита:**
1. `canonicalize()` — разрешение всех символических ссылок
2. Проверка `starts_with(&allowed_dir)` —确保 в разрешённой директории
3. Проверка на абсолютные пути
4. Проверка на `..` компоненты

#### 4. Rate Limiting для таблицы лидеров

**Защита от спама рекордов:**
```rust
// Cooldown 6 секунд между записями (10 записей в минуту)
pub struct Leaderboard {
    pub entries: Vec<LeaderboardEntry>,
    pub last_write_time: Option<Instant>,  // Время последней записи
    pub write_cooldown: Duration,           // 6 секунд cooldown
}
```

**Ограничения:**
- Максимум 10 записей в минуту
- Автоматический cooldown после каждой записи
- Защита от DoS-атак через спам рекордов
- Отключается в тестах через `#[cfg(not(test))]`

#### 5. Безопасная работа с памятью

**Гарантии Rust:**
- Нет висячих указателей
- Нет переполнений буфера
- Нет data races
- Автоматическое управление памятью

#### 5. Обработка ошибок

**Правильная обработка:**
```rust
// Вместо unwrap() используется обработка ошибок
match load_config() {
    Ok(data) => data,
    Err(e) => {
        eprintln!("Предупреждение: {}", e);
        Self::default()
    }
}
```

## 👤 Рекомендации пользователям

### Безопасное использование

#### 1. Установка из проверенных источников

```bash
# ✅ Рекомендуется: установка через cargo
cargo install tetris-cli

# ✅ Альтернатива: сборка из исходников
git clone https://github.com/Githab-capibara/tetris-cli.git
cd tetris-cli
cargo build --release

# ⚠️ Осторожно: сторонние пакеты
# Проверяйте источник пакета перед установкой
```

#### 2. Проверка целостности

```bash
# Проверка хеша бинарника (если доступен)
sha256sum tetris-cli

# Сверка с опубликованным хешем в релизе
```

#### 3. Права доступа

**Рекомендуемые права для файлов конфигурации:**
```bash
# Ограничить доступ к файлам с рекордами
chmod 600 ~/.config/tetris-cli/config.toml
```

#### 4. Обновления

**Регулярно обновляйтесь:**
```bash
# Проверка обновлений
cargo install --force tetris-cli

# Или через git
git pull origin main
cargo build --release
```

### Что делать при подозрении на уязвимость

1. **Немедленно сообщите** мейнтейнеру
2. **Не раскрывайте** детали публично
3. **Дождитесь патча** перед обновлением
4. **Проверьте** файлы конфигурации после обновления
